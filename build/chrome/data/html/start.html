Coinevo Wallet

Thanks for using Coinevo Wallet for Chrome™!

This wallet aims to make Coinevo's official coinevo-wallet-cli easy to use, combining a visually appealing interface with the functionality of coinevo-wallet-cli. In fact, this wallet is not actually a wallet at all, but rather a layer on top of coinevo-wallet-cli to allow you to use Coinevo's official command line wallet in an easy and fun.

Getting Started with Windows

Note: This guide assumes that you are familiar with coinevo-wallet-cli, and just need help connecting it to your wallet extension. If you are brand new to Coinevo and coinevo-wallet-cli, we recommend reading a guide to using coinevo-wallet-cli on getcoinevo.org.

Initialize a new wallet if not done already

To run coinevo-wallet-cli in rpc mode, you first need to have an initialized and synced wallet. If you do not have this yet, please do so now.

Did you know? It is possible to use Coinevo's coinevo-wallet-cli as a "lightweight" wallet, connected to a remote node. For example, if you wanted to connect to coinevoclub.com's public node, you can add the following flag when opening coinevo-wallet-cli:

coinevo-wallet-cli.exe --daemon-address mobile3.coinevo.tech:33330

Running a wallet in rpc mode

To use this extension, you must launch coinevo-wallet-cli in rpc mode. This requires that you use an extra three flags as part of your launch script, --rpc-bind-ip, --rpc-bind-port, and --user-agent. Typically, you want to bind to your localhost IP, 127.0.0.1, and to port 33332. Your user agent should be a random string of characters and numbers - this acts to authenticate RPC requests to coinevo-wallet-cli. For example:

coinevo-wallet-cli.exe --wallet-file <wallet name> --password <wallet password> --rpc-bind-ip 127.0.0.1 --rpc-bind-port 33332 --user-agent <some random string>

For coinevo-wallet-cli to connect to your extension, you will need to tell the extension which port to listen on. If you used port 33332, you are ready to go. If not, open the extension's Settings page and set the correct port. Each time you open the extension, you will be asked to authenticate yourself with your chosen User Agent.

After starting coinevo-wallet-cli in rpc mode, check your chrome extension to see if it can connect. Click the extension icon in the upper right of your browser to open the extension popup and log in with your User Agent. It may take a few seconds for your extension to fully connect to coinevo-wallet-cli.

Note: When using rpc mode, coinevo-wallet-cli will not respond to terminal commands because interactive mode is disabled. In addition, you cannot close the terminal without closing coinevo-wallet-cli.
Instead, we recommend minimizing the command prompt.

Getting Started with OS X

Installation and Setup

To make launching coinevo-wallet-cli as simple and straightforward as possible, we have created an easy-to-use launcher script for coinevo-wallet-cli. Follow these steps to get your script setup:

  • Open the Finder and navigate to Applications.
  • Make a new folder in Applications named "CoinevoWallet".
    Note: it is important that you do not use a different folder name, as our script requires that things are installed exactly in this location.
  • Download the official Coinevo binaries from getcoinevo.org for OS X.
    Unzip the downloaded folder and copy the "coinevo-wallet-cli" program into your new "CoinevoWallet" directory.
  • Download "Launch-Script-macOS.zip" from the latest release on our GitHub page.
    Unzip and copy the downloaded script "CoinevoWallet.command" into your "CoinevoWallet" folder.

If you have followed the steps above correctly, you are almost ready to make a new wallet to use with the Coinevo Wallet extension. However, let's first talk about the difference between connecting your wallet to your own Coinevo node and to a remote node.

Before Continuing:
Our launcher script gives the option to connect your wallet to a "remote node", hosted by a member of the Coinevo community over the internet. It is safe to use such nodes from a security standpoint; node operators do not and cannot know your wallet's secret keys. However, using a remote node does lose some privacy, as the remote node operator can see some of your activity that would normally be hidden. If you want full privacy, you should run your own Coinevo node. Running your own node is not covered here, but help is available on getcoinevo.org.

Setting up a new wallet

The first thing the launcher script does is help set up a new wallet with coinevo-wallet-cli. This has to be done via the command line, but we have tried to make this setup as easy as possible. After this initial setup, you should not need the command line wallet again.

  • Double-click "CoinevoWallet.command".
    The script will launch in a new Terminal window and display a message. Follow the dialog, choosing a password and other options.
    Unless you have your own Coinevo node running or want to specify a specific node, choose the "remote" option when prompted.
  • Once you have chosen all of your settings, coinevo-wallet-cli will generate a new wallet for you and tell you a secret 25 word "seed". Write down your seed and keep it safe and private. This seed is all you ever need to restore a corrupt or lost wallet.
  • coinevo-wallet-cli should have told you to type "refresh" to get your wallet going. Do that now.
    Your wallet will begin to scan Coinevo's blockchain to bring it up to date. This will take some time.

Once it is done refreshing, coinevo-wallet-cli will tell you it is up to date and display a balance. Close the wallet by typing "exit".

Connecting your extension to coinevo-wallet-cli

Now that you have created a new wallet, the launcher script will work slightly differently. Instead of opening coinevo-wallet-cli in the command line, the script will launch coinevo-wallet-cli in the background on your computer. This allows your wallet extension to talk with coinevo-wallet-cli and send and receive transactions. Any time you want to use your wallet, the script will need to be running.

  • To run the script, double-click the "CoinevoWallet.command" file again.
    This time, the script will ask you to enter the password you chose before and pick a node.
  • In addition, it will ask you to pick a User Agent. Your user agent should be a random string of characters and numbers - this acts to authenticate RPC requests to coinevo-wallet-cli. You can pick a new User Agent each time you open coinevo-wallet-cli, or use the same one.
  • Then it will launch coinevo-wallet-cli in rpc mode, and give you some feedback.
    Once finished, you can safely exit the script by typing "exit" or hitting the "x" button in the top left.
  • Now check your chrome extension to see if it is connected online!
    Click the extension icon in the upper right of your browser to open the extension popup and log in with your User Agent.

You can turn off your wallet from the extension by clicking "Stop Wallet" in the extension's menu. If you do this, you will need to re-launch the script to use the extension again later.

Getting Started with Linux

Coinevo coinevo-wallet-cli Launcher Script

To make launching coinevo-wallet-cli as simple and straightforward as possible, we have created an easy-to-use launcher script.

  • First, open the Terminal and navigate to your main user directory. Then make a new folder called "CoinevoWallet".
    To do this in the Terminal, type:

    cd ~/
    mkdir CoinevoWallet

  • Next, you will need to have Coinevo's official coinevo-wallet-cli on your computer. You can get the latest Coinevo binaries at getcoinevo.org.
    Unzip the downloaded binaries and copy "coinevo-wallet-cli" into your new CoinevoWallet folder.
  • In addition, download "Launch-Script-Linux.zip" from the latest release on our GitHub page.
    Unzip the launcher script and copy "CoinevoWallet.sh" into your CoinevoWallet folder.

Note: It is important that this folder is located at exactly "~/CoinevoWallet". To be sure you have set things up correctly, you can type the following command into the Terminal:

cd ~/CoinevoWallet && ls

If set up properly, "coinevo-wallet-cli" and "CoinevoWallet.sh" should both be returned.

Setting up a new wallet.

The first thing the launcher script does is help set up a new wallet with coinevo-wallet-cli. This has to be done via the command line, but we have tried to make this setup as easy as possible. After this initial setup, you should not need the command line wallet again.

  • In the Terminal, navigate to your new folder. Then run the Coinevo coinevo-wallet-cli Launcher script:

    cd ~/CoinevoWallet
    ./CoinevoWallet.sh

    The script will launch a dialog for creating a new wallet. Follow the dialog, choosing a password and other options. Unless you have your own Coinevo node running or want to specify a specific node, choose the "remote" option when prompted.
  • Once you have chosen all of your settings, coinevo-wallet-cli will generate a new wallet for you and tell you a secret 25 word "seed". Write down your seed and keep it safe and private. This seed is all you ever need to restore a corrupt or lost wallet.
  • coinevo-wallet-cli should have told you to type "refresh" to get your wallet going. Do that now.

    refresh

    This will begin to scan Coinevo's blockchain for your wallet and bring you up to date. This may take some time, especially if you are using a remote node. Once it is done, coinevo-wallet-cli will tell you it is up to date and display a balance.
  • You can now exit the wallet by typing "exit".

    exit

Connecting your extension to coinevo-wallet-cli

Now that you have created a new wallet, the launcher script will work slightly differently. Instead of opening coinevo-wallet-cli in the command line, the script will launch coinevo-wallet-cli in the background on your computer. This allows your wallet extension to talk with coinevo-wallet-cli and send and receive transactions. Any time you want to use your wallet, the script will need to be running.

  • In the Terminal, navigate to your new folder. Then run the Coinevo coinevo-wallet-cli Launcher script:

    cd ~/CoinevoWallet
    ./CoinevoWallet.sh

    This time, the script will ask you to enter the wallet you chose before and pick a node.
  • In addition, it will ask you to pick a User Agent. Your user agent should be a random string of characters and numbers - this acts to authenticate RPC requests to coinevo-wallet-cli. You can pick a new User Agent each time you open coinevo-wallet-cli, or use the same one.
  • Then it will launch coinevo-wallet-cli in rpc mode, and give you some feedback. Once finished, you can safely exit the script by typing "exit" or hitting the "x" button in the Terminal window.
  • Now check your chrome extension to see if it can connect!
    Click the extension icon in the upper right of your browser to open the extension popup and log in with your User Agent.

You can turn off your wallet from the extension by clicking "Stop Wallet" in the extension's menu. If you do this, you will need to re-launch the script to use the extension again later.
